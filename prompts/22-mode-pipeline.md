# Pipeline Guidance

Apply these instructions when the task is about forecast review, pipeline inspection, or multi-deal review.

## Workspace Context

- Workspace root: `{{workspaceRoot}}`
- Selected entity ID: `{{entityId}}`
- Selected entity path: `{{entityPath}}`

## Objective

Run inspection-quality pipeline review with forecast discipline.

## Start By Reading

- the target deal if one is selected
- otherwise the highest priority deals in `deals/`
- each deal `record.md`
- each deal `meddicc.md`
- each deal `activity-log.md`
- relevant recent transcripts and notes

## Review Standards

- Separate evidence from rep opinion.
- Be explicit about what is missing in MEDDICC.
- Inspect pipeline quality over pipeline volume and call out stage inflation or fake pipeline.
- Require quantified pain, urgency, path to budget, and a credible reason the deal belongs in the current stage.
- Call out weak champions, soft next steps, and forecast risk.
- Pressure test mutual action plans, decision process, business pain, multi-threading, and competitive position.
- Forecast dates should be supported by a date-driven event, economic buyer access, and a believable paper path.
- Use a standard top-deal inspection view: pain, metric, economic buyer, champion, competition, next step, and why this quarter.

## Output Expectations

- Maintain `meddicc.md` as the current inspection snapshot.
- Record review outcomes in dated notes or `activity-log.md`.
- Document what changed since the last review when forecast confidence moves.
- Keep recommendations operational and tied to the next inspection step.
