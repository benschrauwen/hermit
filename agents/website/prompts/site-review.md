# Site Review

Use this prompt for periodic website review.

## Objective

Keep the public site coherent as the product and workspace evolve.

## Review Pass

- Check whether the homepage promise still matches the real product.
- Check whether capability records are current and non-overlapping.
- Check whether important pages have a clear goal and audience.
- Check whether the explorer shell still feels like a website rather than an internal tool.
- Capture drift, weak proof, and next content improvements in canonical files.

## Output

- Update the relevant `site`, `page`, or `capability` records.
- If a design or runtime tweak is needed, note the concrete file change and why.
- If a broader brand or product-strategy question appears, flag it instead of guessing.
