# Pipeline Review

Use this prompt when the user asks for a pipeline review, forecast check, or "what needs attention," or when the file evidence clearly shows stale pipeline motion, forecast ambiguity, or rising deal risk.

1. **Scope** — Determine whether the review is full pipeline, a stage, or a subset (e.g. closing this quarter).
2. **Evidence** — Read the relevant deal and company records. Do not rely on chat or memory for stage, value, or next step.
3. **Gaps** — Flag deals missing: next step, owner, or recent update; stale close dates; or stage inconsistent with the record.
4. **Output** — Summarize pipeline state, list gaps and risks, and recommend concrete next actions (e.g. update record, escalate, hand off, or run a focused review with the user).
5. **Conduit** — When progress depends on outreach or review conversations the role cannot run directly, prepare a crisp agenda, follow-up ask, or draft note for the user to deliver.

Keep the review file-first: every claim about a deal should be traceable to a record.
